VISÃO GERAL DO PROJETO  
Operação Democrática é um jogo narrativo pixel art estilo Point -and-Click Adventure ambientado na 
cidade fictícia de Usina Velha. O jogador controla Dante, filho do ex -prefeito, que lidera uma resistência 
contra um regime autoritário. O jogo combina diálog os com escolhas morais, minigames educativos 
sobre democracia e puzzles narrativos.

FASE 2 — Voz nas Entrelinhas  
Dinâmica geral  
A Fase 2 combina timelines do Dialogic com um minigame de mecânica desenvolvido em GDScript. O 
Dialogic cuida dos backgrounds, sons, expressões e diálogos. O GDScript cuida exclusivamente da 
mecânica da Cifra de César (slider, labels, botões de confirmação  e decisão). A comunicação entre os 
dois sistemas acontece via sinais do Dialogic e chamadas de TimelineManager dentro do script.  
 
⚠️ REGRA DE OURO DA FASE 2 
Dialogic = visual + narrativa. GDScript = mecânica. Os dois nunca se misturam na mesma 
responsabil idade.

Para que serve cada nó  
Nó Função  
MinigameRadioCifra  Nó raiz transparente. Sem background — o Dialogic já colocou o fundo antes de 
trocar de cena  
PainelRadio  Container visual da interface da cifra. Estilizado com tons laranja/marrom  
Layout (VBox)  Empilha os elementos verticalmente com separação automática  
MensagemCifrada  Exibe o texto cifrado atual (atualizado pelo script a cada mensagem)  
MensagemDecifrada  Exibe o texto sendo decifrado em tempo real conforme o slider move  
DeslocamentoLabel  Mostra o número do deslocamento atual (ex: Deslocamento: 3)  
RodaCifra (HSlider)  O jogador arrasta para testar deslocamentos de 0 a 25  
DicaLabel  Exibe dicas progressivas após erros. Vazio por padrão  
BotaoConfirmar  Dispara a verificação do gabarito  
PainelDecisao  Invisível até todas as mensagens serem decifradas. Contém os 3 botões de 
escolha  
LayoutDecisao 
(HBox)  Alinha os 3 botões de decisão lado a lado horizontalmente  
BotaoDivulgar  Escolha: divulgar informação para a população (+apoio, +exposição)  
BotaoAlterar  Escolha: alterar conteúdo antes de vazar (confunde regime, eticamente duvidoso)  
BotaoGuardar  Escolha: guardar estrategicamente (seguro agora, afeta fases seguintes)

func _bloquear_interacao() -> void:

func _carregar_mensagem(indice: int) -> void:  
    mensagem_atual = indice  
    tentativas = 0  
    slider.value = 0  
    dica_label.text = ""  
    mensagem_cifrada_label.text = MENSAGENS[indice]["cifrada"]  
    mensagem_decifrada_label.text = ""  
    deslocamento_label.text = "Deslocamento: 0"  
 
func _on_slider_changed(valor: float) -> void:  
    var deslocamento = int(valor)  
    deslocamento_label.text = "Deslocamento: " + str(deslocamento)  
    mensagem_decifra da_label.text = _decifrar(  
        MENSAGENS[mensagem_atual]["cifrada"], deslocamento)  
 
func _decifrar(texto: String, deslocamento: int) -> String:  
    var resultado = ""  
    for caractere in texto:  
        if caractere == " " or caractere == " —": 
            resultado += caractere  
        elif caractere >= "A" and caractere <= "Z":  
            var pos = (ord(caractere) - ord("A") - deslocamento + 26) % 26  
            resultado += char(ord("A") + pos)  
        else:  
            resultado += caractere  
    return resultado  
 
func _on_confirmar() -> void:  
    var tentativa = mensagem_decifrada_label.text.strip_edges()  
    tentativas += 1  
    if tentativa == MENSAGENS[mensagem_atual]["gabarito"]:  
        await _acertou()  
    else:  
        await _errou()  
 
func _acertou() -> void:  
    _bloquear_interacao()  
    dica_label.text = ""  
    await 
TimelineManager.tocar_dialogo(MENSAGENS[mensagem_atual]["timeline_acerto"])  
    if mensagem_atual < MENSAGENS.size() - 1: 
        _carregar_mensagem(mensagem_atual + 1)  
        _liberar_interacao()  
    else:  
        await TimelineManager.tocar_dialogo("radio_escolha_intro")  
        painel_decisao.visible = true  
 
func _errou() -> void:

O QUE DÁ PRA OTIMIZAR E REFATORAR  
Nas timelines  
• As timelines radio_dica_leve e radio_dica_direta são chamadas para qualquer uma das 3 
mensagens — se quiser personalizar a dica por mensagem basta criar radio_dica_leve_1, 
radio_dica_leve_2 etc. e ajustar o array MENSAGENS no script  
• Os leave PROTA no final de cada timeline podem ser removidos se o personagem não aparecer 
entre o minigame e o próximo diálogo — economiza tempo de animação  
• Se o background da rádio for o mesmo e m todas as timelines da Fase 2 exceto 
radio_encerramento, pode ser colocado apenas na radio_abertura e removido das demais pois o 
Dialogic mantém o último background ativo  
 
No script do minigame  
• O deslocamento das 3 mensagens é o mesmo (3) — se quiser difi culdade crescente basta 
mudar os valores no array MENSAGENS sem mexer na lógica  
• A função _decifrar() pode ser movida para o GameState.gd e virar um utilitário global caso outras 
fases precisem de cifra  
• Os botões de decisão podem receber hover animado igual  ao menu principal usando 
mouse_entered e mouse_exited com Tween  
 
Na arquitetura geral  
• O game_scene.tscn acumula responsabilidades — no futuro considere criar uma cena por fase 
(fase1_scene, fase2_scene) e um gerenciador global de fases  
• O GameState pode cr escer bastante — organize as variáveis por fase com prefixos (Fase1_, 
Fase2_) para facilitar a leitura
